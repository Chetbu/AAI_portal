## Phase 1 — Foundation

### 1.1 Provision the VPS

**Objective**: A clean Ubuntu server accessible via SSH key only.

**Recommended spec for running ~5-8 POCs simultaneously:**
```
OS:     Ubuntu 24.04 LTS
CPU:    4 vCPU
RAM:    8 GB
Disk:   80 GB SSD
```

**Steps once the VPS is allocated:**

```bash
# 1.1.1 — Connect as root and create a service user
ssh root@<VPS_IP>

adduser aai --disabled-password --gecos ""
usermod -aG sudo aai
echo "aai ALL=(ALL) NOPASSWD:ALL" > /etc/sudoers.d/aai

# 1.1.2 — Copy your SSH key to the new user
mkdir -p /home/aai/.ssh
cp /root/.ssh/authorized_keys /home/aai/.ssh/
chown -R aai:aai /home/aai/.ssh
chmod 700 /home/aai/.ssh
chmod 600 /home/aai/.ssh/authorized_keys

# 1.1.3 — Harden SSH
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/^#\?PubkeyAuthentication.*/PubkeyAuthentication yes/' /etc/ssh/sshd_config
systemctl restart sshd

# 1.1.4 — Firewall
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP  (for Let's Encrypt + redirect)
ufw allow 443/tcp   # HTTPS
ufw --force enable
ufw status verbose

# 1.1.5 — System updates
apt update && apt upgrade -y
apt install -y curl wget git make jq unzip htop
```

**Verify:**
```bash
# From your local machine — can you SSH as aai?
ssh aai@<VPS_IP>
# From your local machine — root login is rejected?
ssh root@<VPS_IP>   # should fail
```

## Phase 2 — Reverse Proxy (Traefik + SSL)

### 2.1 Traefik Static Configuration

```bash
cat > /opt/aai/infrastructure/traefik/traefik.yml << 'EOF'
# =============================================================
# Traefik v3 — Static Configuration
# =============================================================

api:
  dashboard: true
  insecure: false       # dashboard only via routers with auth

log:
  level: INFO           # change to DEBUG when troubleshooting

accessLog:
  filePath: "/var/log/traefik/access.log"
  bufferingSize: 100

# ── Entrypoints ──────────────────────────────────────────────
entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
          scheme: https
          permanent: true
  websecure:
    address: ":443"
    http:
      tls:
        certResolver: letsencrypt

# ── Providers ────────────────────────────────────────────────
providers:
  docker:
    endpoint: "unix:///var/run/docker.sock"
    exposedByDefault: false     # only containers with traefik.enable=true
    network: aai-public
  file:
    directory: "/etc/traefik/dynamic"
    watch: true

# ── Certificate Resolvers ───────────────────────────────────
certificatesResolvers:
  letsencrypt:
    acme:
      email: "${ACME_EMAIL}"    # Note: Traefik does NOT interpolate env vars in YAML.
                                 # We will handle this — see 2.1.1 below.
      storage: /acme.json
      httpChallenge:
        entryPoint: web
EOF
```

#### 2.1.1 — Fix: Traefik doesn't interpolate env vars in its static config

Traefik's YAML config does **not** support `${VAR}` substitution. We solve this with `envsubst` at startup via a wrapper entrypoint.

```bash
# Create a template version
mv /opt/aai/infrastructure/traefik/traefik.yml \
   /opt/aai/infrastructure/traefik/traefik.yml.template

cat > /opt/aai/infrastructure/traefik/entrypoint-wrapper.sh << 'SCRIPT'
#!/bin/sh
set -e

# Substitute env vars into traefik config
envsubst '${ACME_EMAIL}' \
  < /etc/traefik/traefik.yml.template \
  > /etc/traefik/traefik.yml

# Execute the original Traefik entrypoint
exec /entrypoint.sh "$@"
SCRIPT

chmod +x /opt/aai/infrastructure/traefik/entrypoint-wrapper.sh
```

### 3.3 Update docker-compose.yml — Add oauth2-proxy

Add this service block to `/opt/aai/infrastructure/docker-compose.yml`:

```yaml
  # ── OAuth2 Proxy ───────────────────────────────────────────
  oauth2-proxy:
    image: quay.io/oauth2-proxy/oauth2-proxy:v7.6.0
    container_name: aai-oauth2-proxy
    restart: unless-stopped
    environment:
      # Azure AD provider config
      - OAUTH2_PROXY_PROVIDER=azure
      - OAUTH2_PROXY_AZURE_TENANT=${AZURE_TENANT_ID}
      - OAUTH2_PROXY_CLIENT_ID=${AZURE_CLIENT_ID}
      - OAUTH2_PROXY_CLIENT_SECRET=${AZURE_CLIENT_SECRET}
      - OAUTH2_PROXY_OIDC_ISSUER_URL=https://login.microsoftonline.com/${AZURE_TENANT_ID}/v2.0
      
      # Session / cookie
      - OAUTH2_PROXY_COOKIE_SECRET=${COOKIE_SECRET}
      - OAUTH2_PROXY_COOKIE_DOMAINS=.${BASE_DOMAIN}
      - OAUTH2_PROXY_COOKIE_SECURE=true
      - OAUTH2_PROXY_COOKIE_SAMESITE=lax
      - OAUTH2_PROXY_COOKIE_NAME=_aai_oauth2
      - OAUTH2_PROXY_COOKIE_EXPIRE=168h
      
      # Authorization
      - OAUTH2_PROXY_EMAIL_DOMAINS=*
      - OAUTH2_PROXY_AUTHENTICATED_EMAILS_FILE=/etc/oauth2-proxy/allowed_emails.txt
      
      # Behavior
      - OAUTH2_PROXY_HTTP_ADDRESS=0.0.0.0:4180
      - OAUTH2_PROXY_REVERSE_PROXY=true
      - OAUTH2_PROXY_SET_XAUTHREQUEST=true
      - OAUTH2_PROXY_SET_AUTHORIZATION_HEADER=true
      - OAUTH2_PROXY_PASS_USER_HEADERS=true
      - OAUTH2_PROXY_WHITELIST_DOMAINS=.${BASE_DOMAIN}
      - OAUTH2_PROXY_REDIRECT_URL=https://auth.${BASE_DOMAIN}/oauth2/callback
      
      # Skip the "click to sign in" interstitial page
      - OAUTH2_PROXY_SKIP_PROVIDER_BUTTON=true
    volumes:
      - ./oauth2-proxy/allowed_emails.txt:/etc/oauth2-proxy/allowed_emails.txt:ro
    networks:
      - aai-public
    labels:
      - "traefik.enable=true"
      
      # ── Router: serves the /oauth2/ endpoints (login, callback, sign_out) ──
      - "traefik.http.routers.oauth2-proxy.rule=Host(`auth.${BASE_DOMAIN}`)"
      - "traefik.http.routers.oauth2-proxy.tls.certresolver=letsencrypt"
      - "traefik.http.services.oauth2-proxy.loadbalancer.server.port=4180"
      
      # ── Middleware definition: ForwardAuth ─────────────────────────────────
      # Other services reference this as "oauth@docker"
      - "traefik.http.middlewares.oauth.forwardauth.address=http://oauth2-proxy:4180/oauth2/auth"
      - "traefik.http.middlewares.oauth.forwardauth.trustForwardHeader=true"
      - "traefik.http.middlewares.oauth.forwardauth.authResponseHeaders=X-Auth-Request-User,X-Auth-Request-Email,X-Auth-Request-Access-Token"
      
      # ── Error handling: redirect to login page on 401 ─────────────────────
      - "traefik.http.middlewares.oauth-errors.errors.status=401"
      - "traefik.http.middlewares.oauth-errors.errors.service=oauth2-proxy@docker"
      - "traefik.http.middlewares.oauth-errors.errors.query=/oauth2/sign_in?rd={url}"
```

#### 4.1.3 — Server-side health checker script

This runs inside the portal container as a background process. It curls each project's **internal Docker hostname** (no auth needed, no CORS issues) and writes `status.json`.

```bash
cat > healthcheck.sh << 'BASH'
#!/bin/sh
# healthcheck.sh — runs in background inside the portal container
# Reads config.json, checks each project's internal health endpoint,
# writes results to status.json every 30 seconds.

CONFIG_FILE="/usr/share/nginx/html/config.json"
STATUS_FILE="/usr/share/nginx/html/status.json"

while true; do
    # Start building JSON
    RESULT="{"
    FIRST=true

    # Parse projects from config.json using jq
    PROJECTS=$(jq -r '.projects[] | @base64' "$CONFIG_FILE" 2>/dev/null)

    for PROJECT_B64 in $PROJECTS; do
        # Decode project JSON
        SLUG=$(echo "$PROJECT_B64" | base64 -d | jq -r '.slug')
        HEALTH_URL=$(echo "$PROJECT_B64" | base64 -d | jq -r '.healthInternal // empty')

        if [ -z "$HEALTH_URL" ]; then
            continue
        fi

        # Perform health check with 5s timeout
        START_MS=$(date +%s%3N 2>/dev/null || echo "0")
        HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" --max-time 5 "$HEALTH_URL" 2>/dev/null)
        END_MS=$(date +%s%3N 2>/dev/null || echo "0")

        if [ "$HTTP_CODE" -ge 200 ] && [ "$HTTP_CODE" -lt 400 ]; then
            STATUS="up"
        else
            STATUS="down"
        fi

        RESPONSE_TIME=$((END_MS - START_MS))
        NOW=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

        # Append to JSON
        if [ "$FIRST" = true ]; then
            FIRST=false
        else
            RESULT="${RESULT},"
        fi

        RESULT="${RESULT}\"${SLUG}\":{\"status\":\"${STATUS}\",\"httpCode\":${HTTP_CODE},\"responseTimeMs\":${RESPONSE_TIME},\"lastCheck\":\"${NOW}\"}"
    done

    RESULT="${RESULT}}"

    # Write atomically (write to temp file, then move)
    echo "$RESULT" > "${STATUS_FILE}.tmp"
    mv "${STATUS_FILE}.tmp" "$STATUS_FILE"

    sleep 30
done
BASH

chmod +x healthcheck.sh
```

```bash
BACKUP_FILE="${BACKUP_DIR}/aai-backup-${DATE}.tar.gz"

mkdir -p "$BACKUP_DIR"

echo "[backup] Starting backup: ${DATE}"

# Backup everything except Docker volumes (those are ephemeral for POCs)
tar czf "$BACKUP_FILE" \
    --exclude='*/node_modules' \
    --exclude='*/__pycache__' \
    --exclude='*.pyc' \
    --exclude='*/venv' \
    /opt/aai/shared.env \
    /opt/aai/infrastructure/ \
    /opt/aai/projects/

echo "[backup] Created: ${BACKUP_FILE} ($(du -sh "$BACKUP_FILE" | cut -f1))"

# Keep only last 7 backups
ls -t ${BACKUP_DIR}/aai-backup-*.tar.gz | tail -n +8 | xargs -r rm --
echo "[backup] Cleanup complete. Current backups:"
ls -lh ${BACKUP_DIR}/aai-backup-*.tar.gz
BASH

chmod +x /opt/aai/infrastructure/scripts/backup.sh

# Add cron job: daily at 3am
(crontab -l 2>/dev/null; echo "0 3 * * * /opt/aai/infrastructure/scripts/backup.sh >> /var/log/aai-backup.log 2>&1") | crontab -
```

### 6.3 Resource Limits

Update **every project's `docker-compose.yml`** to include memory limits:

```yaml
services:

    # ... existing config ...
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '1.0'
        reservations:
          memory: 128M
          cpus: '0.25'
```
